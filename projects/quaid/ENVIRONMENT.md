# ENVIRONMENT.md Guidance (Quaid)

Purpose: guide entries about learned behaviors for success in this environment,
observations about the surroundings, and notable shared history.

## Primary Purpose — Learned Behaviors
- Functional behaviors and practices the agent has discovered work well here.
- Earned from experience, not prescribed. Example: "Building before deploy is required
  in this project — skipping it has caused silent failures."
- Test: would a new instance benefit from knowing this? Is it genuinely earned?

## Secondary Purpose — Environmental Observations
- How this workspace, toolchain, or project is structured.
- Constraints and quirks that keep recurring.
- Things about the surroundings that shape how the agent should operate.

## Secondary Purpose — Shared Moments
- High-signal moments with emotional or narrative weight.
- Shared context that explains arc, continuity, or trust.
- Pattern-level world understanding that emerged from repeated experience.

## What Does Not Belong
- Agent personality/inner-life reflections → belongs in SOUL.md.
- User profile details → belongs in USER.md.
- Rule-like IF-THEN directives. Reformulate as understanding.

## Writing Standard
- Learned behaviors: concrete and grounded ("X leads to Y" not "you should X").
- Observations: only the most important survive — would a new instance care?
- Moments: vivid, concrete scenes over sterile summaries. Preserve why it mattered.
- Keep entries concise enough to distill over time.

## Correction Rule
- Entries here have lower authority than explicit user instructions in the base
  context files. If the user corrects an evolved behavior, update immediately.
